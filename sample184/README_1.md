3. 関数の基本
関数は処理をまとめるテンプレート

書いただけでは実行されない → 関数名() で呼び出す必要あり

同じ処理を何度も使えるので 効率的・見やすい

4. 関数のメリット
修正が一括反映される → 例えば「10回繰り返し」を「5回」に変更すると、呼び出し部分すべてに反映される

再利用性が高い → 何度も同じ処理を使う場面で便利

5. 引数（パラメータ）
関数に値を渡すことで 柔軟に処理を変えられる

例: numCount(max) として max 回繰り返す

初期値を設定できる → よく使う値を省略可能 （例: function numCount(max = 10)）

6. 戻り値（return）
関数の中で return を使うと 計算結果などを返せる

画面に直接表示するのではなく、他の処理に使うための値

例: 0〜9の合計を計算して return value

戻り値は 関数を実行しないと得られない → const result = numCount();

7. 注意点
関数呼び出しの 丸括弧 () を忘れない

戻り値を変数に入れると、他の処理（例: alert(result)) に活用できる

まとめ
関数は 処理をまとめて再利用する仕組み

引数で柔軟性を持たせ、戻り値で結果を活用できる

「何度も同じ処理を書くのは面倒だな」と思ったら関数化するのがベスト


✅ 関数の戻り値は記述は必須なのか？
結論から言うと 戻り値（return）の記述は必須ではありません。 
JavaScriptの関数は、return を書かなくても動作します。
その場合、関数の戻り値は 自動的に undefined になります。

具体例
js
function greet() {
  console.log("こんにちは");
}

const result = greet(); 
console.log(result); // → undefined
greet() は return を書いていないので、戻り値は undefined

ただし console.log("こんにちは") は実行されるので、画面には「こんにちは」と表示される

✅ 戻り値が必要な場面
計算結果を使いたいとき

js
function add(a, b) {
  return a + b;
}
const sum = add(3, 5); // sum = 8
他の処理に値を渡したいとき

例: APIから取得したデータを返す

例: 条件判定の結果を返す

✅ 戻り値が不要な場面

画面に表示するだけ

ログを出すだけ

副作用（DOM操作やデータ更新）を目的とする処理

まとめ
return は 必須ではない

書かなければ undefined が返る

値を使いたいときだけ return を書く

【深堀り】
処理を関数化せずに、そのまま記述するとプログラムの順次処理によりそのまま実行され、関数するとその処理の実行タイミングや実行箇所が容易にコントロールできるメリットがある。

✅ 関数化しない場合
プログラムは 上から下へ順番に実行されるため、書いた瞬間に処理が走ります。

つまり「書いた場所＝実行場所」になり、制御はしづらいです。

✅ 関数化した場合
関数は「処理のまとまり」を 名前付きで保存しておく仕組み。

書いた時点では実行されず、呼び出したタイミングで初めて実行されます。

これにより:

いつ実行するかを自由に決められる

どこで実行するかを選べる

同じ処理を 何度でも再利用できる

例でイメージ
js
// 関数化しない場合
console.log("処理A");
console.log("処理B");
// → 順番に必ず実行される
js
// 関数化した場合
function doProcess() {
  console.log("処理A");
  console.log("処理B");
}

// 好きなタイミングで呼び出せる
doProcess(); // 今ここで実行
// doProcess(); // もう一度実行も可能
まとめ
関数化しない → 書いた順に必ず実行

関数化する → 実行タイミング・場所をコントロールできる

だから関数は「処理の再利用」と「実行の自由度」を与えてくれる仕組みです。
